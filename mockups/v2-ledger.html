01 The gap

Generative AI doesn’t fail loudly. It fabricates fluently.

Tone, safety, and PII filters judge style and policy. None of them check whether a claim is true for this merchant. When a model writes “you’ll be live by Friday” for an account that has no go-live date, every conventional guardrail waves it through.

Why existing guardrails structurally miss it

A guardrail evaluates the text in isolation — toxicity, jailbreak patterns, regex for PII. Faithfulness is not a property of the text; it is a relation between the text and a record.

It can only be checked by reconciling each asserted fact against the structured source of truth — the data row, not a retrieved RAG passage. That reconciliation is exactly what a style/safety filter does not do.

03 How it works

Deterministic first. The model is checked, not trusted.

Five stages. Cheap, exact checks run before the expensive one; a second, independent model runs last; a person signs off.

Why two checks, and why a different model for the second

The deterministic gate verifies declared claims forward (claim → field) and is byte-for-byte reproducible — it’s the fast, exact floor. But a model can assert a fact in prose without declaring it as a claim; the gate can’t see that.

The faithfulness judge closes that gap from the other direction (prose → data entailment). It runs on a cross-family model so the verifier isn’t grading its own family’s output — maker ≠ judge at the model layer. It’s tuned recall-favoring: a false hold costs a glance; a missed fabrication costs trust.

05 Measured, not asserted

Honesty is part of the control.

The deterministic core is locked.
It reproduces a byte-identical reference output across every record — a differential test is the floor the whole pipeline stands on.

The judge is calibrated, not assumed.
