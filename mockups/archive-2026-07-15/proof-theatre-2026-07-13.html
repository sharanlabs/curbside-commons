Act 02

How the decision is made

Plain words first. Exact receipts behind them.

The system does not ask a viewer to trust a score. It shows what was claimed, which record was consulted, which rule decided the difference, and how severe it is.

01

Start with the record

The merchant system-of-record defines the price, availability, identity, and other checkable facts.

02

Read each serving copy

The same underlying catalog may be expressed through multiple agent-facing surfaces. Each copy is examined independently.

03

Compare deterministically

Explicit rules compare checkable claims with their matching reference rows. The same fixture produces the same report.

04

Attach the receipts

Every finding carries the claim, reference row, rule or clause, and severity. Technical detail is available without obscuring the plain-language result.

05

Keep action human-gated

The proof supports review; it does not quietly turn a finding into an external action.

Act 05

The honest edge

The limitations belong in the product story.

This is a simulated, run-on-demand prototype built to demonstrate verification rigor. It is not evidence of market adoption, production impact, or a continuously operated service.

Recorded miss

The first classifier run did not earn the label.

It scored 20/21 but missed one conjunctive recall floor. The result remained public and the label stayed deferred.

Simpler lane kept

The retrieval experiment did not beat its baseline.

The hybrid lane missed its floors, so the simpler BM25 reference lookup remains the advisory path.

Protected default kept

Fuzzy matching did not outperform exact matching.

The ensemble tied the normalized-exact baseline while missing recall. Exact matching stays the default.
