02 Method

How a verdict works

The claim, the rule, and the record stay together.

Every verdict keeps three things attached: what was claimed, the merchant record it was checked against, and the explicit rule that compares them. The result comes from that rule, not from a model's judgment. The same inputs and rule produce the same result, so the conclusion can be retraced instead of accepted on faith.

No language model runs inside the verdict. A model may later draft a plainer explanation, but it never decides the result.

04 Boundaries

Where it stops

When the evidence runs out, the verdict stops too.

Some questions need evidence beyond what was supplied. When a rule depends on a value that was never provided, Curbside Commons marks the case unresolved and says so plainly. It does not fill the gap with a likely answer or turn an absence into a fact. The rule that could not be settled is named alongside the evidence it was missing. That restraint is the point. An explicit unknown is more honest than an invented answer.

External evidence Unresolved

Six of the 17 fee-cap rules depend on evidence beyond the supplied statement. Those cases stay unresolved rather than being forced into pass or fail.

Structure is not truth Reported apart

Schema and protocol conformance establish a valid shape. They do not establish merchant truth, so the two results are reported separately.

Only the inputs given No inference

The verifier evaluates the inputs it receives. It does not infer missing facts, completeness, or freshness, and it does not guess a value that was never supplied.

05 Close

Trust, with receipts

Make the conclusion as inspectable as the claim.

A claim should not be trusted just because it looks complete. A verdict should not be trusted just because it sounds certain. Curbside Commons keeps the claim, the rule, and the merchant's record together, so anyone can retrace the conclusion instead of taking it on faith — the same inputs and the same rule reach the same result.
