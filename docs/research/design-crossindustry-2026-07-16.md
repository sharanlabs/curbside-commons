## D1 — Museum / exhibition (how institutions stage one precious object + its provenance)
References (numiko.com/insights/best-museum-and-gallery-websites-2026, fetched 2026-07-16; the
listed sites named there):
- **Rijksmuseum** — art as the focal point, "simple information architecture," stories told
  around single objects. **V&A** — object-centric model: every object links to its exhibitions,
  events, media; "detailed imagery with clear contextual cues such as 'on display' indicators."
- **Nordiska Museet** (2025 Webby + Lovie) — each exhibition gets its own adapted palette drawn
  from the physical interior; scroll-responsive framing devices.
- **MIT Museum** — geometric "peepholes" of color blocking that reveal the object; curiosity-first.
- **Nottingham Contemporary** — deliberate brutalist subversion: proof that a museum register can
  be DISTINCT, not polite.

**Principle:** a museum page treats the object as singular — one artifact, full provenance chain,
institutional labeling (accession number, date, origin, condition). The label typography IS the
authority. **Personalize:** our held claim IS the museum object. Give the 2150-specimen an
accession-card treatment: a catalogued exhibit with provenance chain (served claim → statement
line → rule → verdict), each hop a labeled, dated entry. BORROW (zero risk, pure IA/typography).
Per-chapter adapted palettes (Nordiska) → ADAPT: per-stage accent temperature inside one system.
Peephole reveals → REJECT (curiosity-gating conflicts with opens-complete).

## D2 — Luxury / haute horlogerie (staging, restraint + spectacle)
References (search layer, 2026-07-16: iiad.edu.in "The UI/UX of Luxury", muffingroup.com luxury
examples, kijo.london + builtwithorbit.com watch-site teardowns; Tier 2–4, principles corroborated
across ≥3 sources):
- Luxury motion is "choreography, not a dance party" — slow, smooth, purposeful; enrichment, not
  attention-grabbing. Serif display (Didot/Bodoni genre) + generous whitespace + restrained
  palette (black/white/gold/deep navy) = the "feels expensive" register.
- Watch product pages: macro detail photography, 360° object staging, heritage storytelling
  sections; the object is lit like jewelry against a void.

**Principle:** premium = fewer elements, each finished to a higher tolerance; the object gets
LIGHT, the chrome disappears; motion has a watchmaker's tempo (slow ease, no bounce).
**Personalize:** treat the arithmetic itself as the jewel — the ×100 delta staged with macro-detail
care (large, lit, isolated on a void band), everything else recedes. ADAPT (our "materials" are
digits and rules, not gold). Heritage-timeline sections → REJECT (we have no heritage; honesty).

## D3 — Automotive (cinematic chapters, spec-sheet authority)
References (search layer 2026-07-16: azurodigital.com 2026 automotive teardown,
schweitzerdesigns.com 2026 luxury-launch analysis, Porsche configurator cited across sources;
Tier 2–4 corroborated):
- 2026 launch tier = narrative-driven sequences over turntable renders; chaptered scroll
  storytelling; "futuristic, premium, built for motion."
- The configurator pattern: the ONE object persists on stage while the user changes its state;
  every change re-renders the same object — never a page change.
- Spec-sheet presentation: engineering numbers presented as typography-first tables with unit
  discipline (kW / Nm / 0–100), monospaced alignment, no decoration.

**Principle:** persistent-object staging — the product never leaves the stage; the story moves
around it. Spec-numbers ARE the luxury copy. **Personalize:** the EvidenceBench as configurator:
one persistent specimen (the claim), stage-switching relights the SAME object (step-through =
"trim levels" of the examination). BORROW persistent-object; ADAPT spec-sheet type discipline for
the receipt (unit-true mono columns). Dark neon-wave aesthetic → REJECT (slop-adjacent per the
session-20 digest; our ground is paper-white).

## D4 — Editorial forensics (evidence + conclusion in one frame)
References: Forensic Architecture methodology (en.wikipedia.org/wiki/Forensic_Architecture,
fetched 2026-07-16 — the FA site itself served a JS shell to the fetcher, direct read UNVERIFIED;
forensic-architecture.org/methodology/visualization is the primary source to re-read in a browser):
- Video synchronization + geolocation: "incidental visual details become evidentiary."
- 3D reconstruction as a memory aid: testimony reconstructed INSIDE the model.
- Multi-forum legibility: the same investigation is presented to courtrooms AND museums —
  "evidence presentation itself as politically meaningful."
- Timelines correlate multiple evidence sources temporally and geographically (Pattrn/Gaza
  Platform pattern).

**Principle:** the strongest evidentiary UI SYNCHRONIZES sources — the viewer sees the raw
material and the derived conclusion in one registered frame, never on separate pages; time is the
spine. **Personalize:** this is our genre. The bench's step-through = FA's synchronized timeline:
each step lights the statement line (raw source) AND the rule clause (instrument) AND the
arithmetic (derivation) together — one registered frame per hop. BORROW (the deepest structural
borrow of the sweep). The courtroom/museum dual-register → BORROW as copy stance: the same
receipt must read to an auditor and to a stranger.

## D5 — Cinema / title design (futuristic typographic register)
References: artofthetitle.com/titles/2025 (fetched 2026-07-16 — thin year-page: Babystar, Duster;
the deep archive is the reference, flagged partial), envato/printmag title-typography trend pieces
(search layer 2026-07-16):
- Title typography moved from information to storytelling device; futuristic = minimalist + sleek,
  precision-spaced, letter-by-letter arrival, not chrome-and-glow.

**Principle:** a title sequence establishes register in seconds through TYPE IN MOTION alone —
arrival order, tracking, and pacing carry the drama. **Personalize:** chapter eyebrows and the
hero claim can ARRIVE like title cards (letter/word-staggered, WAAPI, reduced-motion = instant),
giving the futuristic register without any decorative object. ADAPT. Full-bleed video/audio
sequences → REJECT (static export, zero external, honesty).

## D6 — Instrument / aerospace HMI (precision = trust)
References (search layer 2026-07-16: e3displays.com aerospace HMI, ansys.com futuristic-cockpit
HMI, aufaitux.com HMI best practices; Tier 2–3):
- Avionics displays: "every piece of information presented with precision, clarity, and
  reliability"; thoughtful design against error IS the trust surface; cleaner digital aesthetic,
  colorful indicators used sparingly and semantically.

**Principle:** an instrument earns trust by REGISTRATION — aligned scales, unit labels, state
lights with fixed meanings, nothing decorative pretending to be data. **Personalize:** the
verdict/tally chrome as instrument cluster: fixed-meaning state lamps (VIOLATION ember / PASS
graphite / HELD gold), registered baselines, unit-disciplined figures. The site already leans
instrument ("Light Gauge", "registered measure") — this sweep says: commit fully, it is the
distinctive register dev-tool sites cannot fake. BORROW.
