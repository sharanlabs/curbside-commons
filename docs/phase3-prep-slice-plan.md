## Why this slice exists (the eval-coverage finding — unchanged from draft 1)

The build-process audit named the pre-Phase-3 gate as two hygiene items (`out/` log policy; enforcement-hooks decision). A direct read of the eval code shows a more important gap:

- **T-002's "MERCHANT 20/20" scores only deterministic fields.** All 14 `GOLDEN_MERCHANT_FIELDS` ([scripts/eval.py:25](../scripts/eval.py:25)) are pipeline outputs that do not come from the draft generator: formula `risk_score`, **source-echoed** `risk_level` ([scripts/pipeline.py:294](../scripts/pipeline.py:294)), STEP_MAP blocker/action, and the eligibility/send-gate booleans.
- **Phase 3 replaces only `make_draft()`** ([scripts/pipeline.py:112](../scripts/pipeline.py:112)) — the generated *text* (`risk_explanation`, `blocker_summary`, `draft_subject`, `draft_body`). **No golden field observes that text.** A Gemini swap leaves "MERCHANT 20/20" unchanged by construction.
- **The guardrail 45/45 is bounded:** 11 hand-written positives the regex was built to catch (regression parity) + 25 clean over-flagging cases + 5 stub-clean. The guardrail is **never tested against model-generated adversarial text**.

**Conclusion:** the current baseline is strong *regression armor for the deterministic core*, but **not yet a measuring stick for the surface Phase 3 changes**. Phase 3's trigger — "the model meets or beats the baseline behind the guardrails" — is not currently checkable. T-003 makes it checkable **before** the model lands.

### T-003·S2 — Generator-agnostic draft contract + eval, on the v1 lane (the core)
