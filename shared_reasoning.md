Do the same: our own TS/ajv validation of the published UCP JSON Schemas at runtime (UCP mandates client-side schema fetch+composition anyway — machine-checkable, per the ADDENDUM primary read), with the official `ucp-schema` CLI run in CI as the conformance oracle ("agrees with the official validator on N fixtures" is a stronger credibility line than "shells out to it"). nekuda becomes a cited see-also + optional CI cross-check, not a dependency.
